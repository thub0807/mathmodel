# 国赛一等奖论文模式 (winning_patterns)

> 从公开渠道 (GitHub `personqianduixue/Math_Model`、`zhanwen/MathModel`、教育部数学建模优秀论文展廊、国赛官方公布的获奖论文集) 抽取的**定量共性**。这些是相关性而非因果关系,但被一致性触发为 rubric 项,可以显著拉高基线分。

---

## 1. 摘要 5 段式骨架 (摘要 = 30% 总分,最高杠杆)

一等奖摘要近乎严格遵循以下 5 段式,顺序不可乱:

```
[段 1 — 背景与问题定义] 100-150 字
本文针对 <核心问题描述>,综合考虑 <2-3 个关键约束/因素>,
建立了 <你的命名模型族> 模型,通过 <求解方法> 求解,得到了
<可量化的总成果概述>。

[段 2 — 任务拆解] 50-100 字  
针对问题一/问题二/问题三,本文分别建立了 <模型1>、<模型2>、<模型3>。

[段 3 — 各小问详细方法 + 数值结果] 250-400 字 (主体,占摘要 50%)
针对问题一,本文 <做了什么> ... 得到 <定量结果数值>。
针对问题二,本文 <做了什么> ... 得到 <定量结果数值>。
针对问题三,本文 <做了什么> ... 得到 <定量结果数值>。

[段 4 — 灵敏度 + 模型评价] 80-120 字
对模型进行了灵敏度分析,在 <参数> 在 ±X% 范围内变化时,
结果稳定 / 误差小于 X%,验证了模型的稳健性。

[段 5 — 创新与推广] 60-100 字  
本文的创新点在于: 1) <X> 2) <Y> 3) <Z>。
所建模型可推广至 <相关场景>。

关键词: <4-6 个,每个不超 8 字>
```

**字数总计**: 600-900 字 (一等奖均值 ~750)。
**定量结果数量**: ≥3 个 (最低)。
**关键词**: 4-6 个,首个一定是问题相关,后面是方法相关。

---

## 2. 正文章节结构 (一等奖标准目录)

```
1. 问题重述                       1-2 页    (重述 + 自己语言提炼)
2. 问题分析                       2-3 页    (分总分式)
   2.1 问题一分析
   2.2 问题二分析
   ...
3. 模型假设                       0.5-1 页  (3-7 条,每条带支撑)
4. 符号说明                       0.5-1 页  (表格)
5. 模型的建立与求解               12-16 页  (主体,占论文 60-70%)
   5.1 问题一: <模型族名称>
       5.1.1 模型建立
       5.1.2 模型求解
       5.1.3 结果分析与可视化
   5.2 问题二: ...
   ...
6. 灵敏度分析与稳健性检验         2-3 页
7. 模型评价与推广                 1-2 页
   7.1 模型优点
   7.2 模型缺点  
   7.3 模型推广
8. 参考文献                       0.5-1 页  (国赛要求 ≥10 条,GB/T 7714 格式)
附录                              不计入页数
   A. 程序代码 (Python/MATLAB)
   B. 计算结果详表
   C. 求解过程截图
```

**总页数**: 22-25 页 (国赛页数上限 25,一等奖通常用满 22-24)。
**正文字数**: 18000-25000 字。
**公式编号**: 60-100 个 (一等奖均值 ~80)。

---

## 3. 图表密度 (评委 30 秒判断质量的视觉信号)

每个子问题最低配置:
- **流程图 1 张**: 算法/求解流程
- **结果图 ≥1 张**: 主要数值结果可视化
- **灵敏度/对比图 1 张** (子问题不强求,全局必须)
- **数据表 1-2 张**: 输入参数 / 输出结果

**全文图表总数**: 一等奖均值 18-25 个 (图 60%,表 40%)。
**图表必须双栏并排**: 节省页面、信息密度更高。

**配色与可读性**:
- 用 matplotlib + seaborn / TikZ,**不要** Excel 默认配色
- 字号 ≥9pt (打印后清晰)
- 公式用 LaTeX,**不用截图**

---

## 4. 模型命名约定 (创新性的廉价信号)

一等奖几乎不用 "AHP 模型"、"BP 神经网络模型" 这种纯textbook名。即使本质是经典模型,也起一个**改进/混合名**:

| beginners 命名 | 一等奖命名 |
|---------------|----------|
| AHP 模型 | 动态权重 AHP-熵权法组合评价模型 |
| BP 神经网络 | 改进型 BP-LSTM 混合预测模型 |
| 灰色预测 | 基于残差修正的 GM(1,1)-马尔可夫预测模型 |
| 整数规划 | 基于二阶锥松弛的混合整数规划模型 |
| SIR 模型 | 考虑潜伏期的改进 SEIR 传染病动力学模型 |
| 蒙特卡罗 | 基于拉丁超立方抽样的蒙特卡罗稳健性仿真 |

模式: `<改进/限定词> + <核心模型> + <复合方法>`。即使 "改进" 只是加了个权重函数,也要起名。

**rubric 触发**: 阶段 3 模型选型时,要求每个子问题的模型至少 2 个修饰词。

## 5. 子问题间的"复用链" (一等奖必备,beginners 必缺)

一等奖论文中,Q3 在某处会**显式调用** Q1 或 Q2 的结果:

```
"将问题一中得到的最优配比 X = (...) 作为本节模型的输入,
 进一步考虑 ... 约束,得到 ..."
```

**评委视角**: 这显示了对题目的整体理解,而非各做各的。

**rubric 触发**:
- Stage 7: `traceability.md` 和 `final_results.md` 必须明确写出是否依赖上游问题结果
- Stage 9: 检查后续子问题是否引用前序结果 (若题目本身允许)

---

## 6. 假设的"支撑度"

一等奖每个假设都有支撑(参考文献/数据观察/物理意义),**不写无支撑假设**。

模式:
```
假设 i: <内容>。
依据: <文献 [N] 表明...> / <对附件数据观察...> / <从物理意义考虑...>。
```

beginners 往往写: "假设车辆匀速行驶。" (孤立、无依据)。

---

## 7. 灵敏度分析的深度 (区分一等奖与二等奖的关键)

| 等级 | 灵敏度做法 |
|------|----------|
| 三等奖 | 不做或仅口头提"模型稳健" |
| 二等奖 | 一个参数 ±10% 变化,看结果变化 (OAT,one-at-a-time) |
| **一等奖** | **多变量联合扰动** (≥3 参数同时变,如拉丁超立方/Sobol),给出**定量稳健区间** |

报告格式:
```
表 N. 多变量联合灵敏度分析结果
参数组合 | 扰动幅度 | 输出指标变化 | 稳健性结论
(α,β)   | ±5%     | 输出 ±2.3%   | 极稳健
(α,β,γ) | ±10%    | 输出 ±7.8%   | 较稳健
```

## 8. 模型评价节的"真自我批判"

一等奖的"模型缺点"是**真的**缺点,不是套话:

**套话 (扣分)**:
> "本模型有一定的简化,未考虑某些因素。"

**真批判 (加分)**:
> "本模型假设需求服从泊松分布,但附件 3 数据的方差/均值比为 1.8,
> 暗示存在过度离散现象。若改用负二项分布,在 Q2 的预测精度可能进一步提升 3-5%,
> 但求解器从 cvxpy 需切换到 GAMS,时间成本上升。"

**rubric 触发**:
- Stage 9: 模型评价中 ≥3 条缺点,每条必须 (a) 命名具体方法 (b) 估算改进空间 (c) 说明代价

---

## 9. 摘要中的"亮点信号词"

一等奖摘要倾向于使用以下信号词组合:

- **创新表述**: "本文创新性地..." / "首次将...应用于..." / "提出了..."
- **定量结果**: "误差仅 X%" / "精度达到 X%" / "运行时间缩短 X 倍"
- **稳健性**: "在...扰动下保持稳定" / "鲁棒性强"
- **推广**: "可推广至..." / "为...提供了新思路"

不要堆砌(每段一个即可),但**完全不用** = 摘要语气平淡 = 减分。

---

## 10. 附录代码的呈现 (常被忽视的 5 分)

一等奖附录代码:
- 有简短说明 (3-5 行注释开头) 标明此段对应论文哪一节
- 缩进、命名规范
- 中文注释完整 (不依赖英文)
- 删除了调试代码 / print 语句 / 失败实验
- 总行数 50-300 行 (代码量与方法复杂度匹配)

beginners:
- 几百行未整理 ipynb 直接 print
- 注释中英混杂、含 TODO

## 与 rubrics.md 的对应

本文件是 **principles** (国赛一等奖共性观察), `rubrics.md` 是 **scoring criteria** (各阶段评分量表)。
rubrics.md 已通过 inline `(see winning_patterns §N)` 引用本文件具体段落, **不在此重复列出**。
anti_patterns.md 各反模式条目也按 `(对应 winning_patterns §N)` 反向引用。

实测分布数据 (从 91 篇真国赛获奖论文烘焙) 见 `empirical_distribution.md`, 用于把本文件的预设阈值 (e.g., "字数 600-900") 替换为实测 p25/p50/p75。

更细的句式锚点、命名变体和格式模式已拆分到 `distilled_phrases.md` / `distilled_naming.md` / `distilled_formats.md` (from distilled, see distilled_*.md), 这里不再重复。

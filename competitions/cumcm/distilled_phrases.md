# 蒸馏后的句式模板

> 下面不是连接词清单, 而是可以直接喂给 LLM 的段落级骨架。
> 每条都尽量控制在 1-3 句, 带 `<填X>` 占位符, 并注明位置与功能。

---

## 摘要段1｜背景与总述

- 模板 1: “本文针对<问题领域>中的<核心子问题, 30-50 字>, 在综合考虑<约束1>、<约束2>和<约束3>的基础上, 建立了<命名变体模型>, 并通过<求解方法>得到<总体定量成果>。”
- 模板 2: “围绕<现实场景>下<关键任务>, 本文从<对象维度>与<机制维度>两个角度展开建模, 先将原问题拆分为<子任务A>与<子任务B>, 再统一为<模型族>求解。”
- 模板 3: “针对题目给出的<数据资源>与<业务约束>, 本文采用<方法组合>刻画<关键关系>, 在<假设条件>下推导出<核心公式或判别准则>, 最终实现<目标>。”
- 模板 4: “本文面向<问题场景>构建<改进/混合/动态>模型, 通过<算法>对<决策变量>进行联合优化, 并以<定量指标>作为结果评价标准。”
- 模板 5: “为解决<问题域>中<核心痛点>, 本文将<理论工具A>与<理论工具B>进行耦合, 形成<命名变体>, 从而对<结果对象>给出可解释的定量结论。”
- 模板 6: “从工程/业务角度看,<问题>的关键不在于单点求值, 而在于<多个约束>共同作用下的整体决策, 因此本文采用<模型>进行统一刻画。”

## 摘要段2｜任务拆解

- 模板 1: “针对问题一, 本文首先识别<输入>与<输出>的映射关系, 再将<语义对象>转化为<数学对象>, 构建<模型1>。”
- 模板 2: “对于问题二, 在问题一结果的基础上, 我们进一步引入<新增因素>, 使模型由<基础版>升级为<增强版>。”
- 模板 3: “进一步地, 问题三需要同时考虑<因素A>、<因素B>与<因素C>, 因此本文采用<统一框架>将三类约束并入同一模型。”
- 模板 4: “问题四与前述子问题不同, 它更强调<分场景/分层/分阶段>分析, 因而本文将其拆分为<分支1>与<分支2>分别处理。”
- 模板 5: “上述三个子问题并非相互独立: <上游结果>将作为<下游模型>的输入, 该复用链能够保持前后章节的一致性。”
- 模板 6: “就任务逻辑而言, 问题一解决的是<定义/估计>, 问题二解决的是<决策/优化>, 问题三解决的是<检验/推广>, 三者共同构成完整闭环。”

## 摘要段3｜方法与结果

- 模板 1: “针对问题一, 本文建立<模型名>, 通过<算法>求解后得到<结果1>; 与基线<方法>相比, 该结果提升/降低了<百分比>。”
- 模板 2: “针对问题二, 在<约束>下引入<新机制>, 得到<结果2>; 该结论说明<业务含义>。”
- 模板 3: “针对问题三, 本文采用<数值方法/仿真器/回归模型>, 输出<指标>; 误差控制在<范围>内, 与理论或实验结果一致。”
- 模板 4: “在全局层面, 通过<统一模型>可同时解释<现象A>与<现象B>, 其总体性能指标达到<数值>, 展现出较好的可解释性与计算效率。”
- 模板 5: “结果表明, 所建模型在<场景1>、<场景2>与<场景3>中均保持同一趋势, 这从侧面验证了建模假设的合理性。”
- 模板 6: “对结果进一步分析可知, <关键参数>并不是单独起作用, 而是通过<耦合机制>共同影响<输出指标>, 因而需要联合调节。”

## 摘要段4｜稳健性与敏感性

- 模板 1: “为评估模型对参数变化的敏感程度, 本文对<参数组>进行<单因素/联合>扰动, 结果显示在<范围>内输出波动较小。”
- 模板 2: “在<±5%/±10%/±20%>三档扰动下, <关键指标>的偏差始终控制在<阈值>以内, 说明模型具有较强稳健性。”
- 模板 3: “进一步的灵敏度分析表明, <参数A>对结果影响最大, <参数B>次之, 而<参数C>的影响相对有限。”
- 模板 4: “当<临界条件>被突破时, 最优解会切换到<备用方案>, 这一现象为实际部署提供了边界提醒。”
- 模板 5: “通过<采样方法>构造的联合扰动实验还显示, 模型输出的排序关系保持不变, 因而鲁棒性较强。”
- 模板 6: “综上, 本文不仅给出了单次求解结果, 还明确了结果对参数变化的响应区间, 使结论更具有可复现性。”

## 摘要段5｜创新与推广

- 模板 1: “本文的创新点在于<创新点1>、<创新点2>与<创新点3>的组合, 使模型从<传统版本>升级为<改进版本>。”
- 模板 2: “与现有方法相比, 本文不仅保留了<优点>, 还补足了<缺陷>, 因而更适合<应用场景>。”
- 模板 3: “所建模型可进一步推广至<场景A>、<场景B>等问题, 只需替换<参数/约束/目标>即可复用。”
- 模板 4: “在工程实现上, 本文形成的建模流程具有较强迁移性, 可为<同类问题>提供一套可直接复用的分析范式。”
- 模板 5: “因此, 本文的价值不只在于解决当前题目, 更在于提供了一个可扩展的<方法框架>, 便于后续加入更多约束或数据源。”
- 模板 6: “总的来说, 本文在<问题域>中实现了<效果>, 并通过<定量指标>证明了方法的有效性与可推广性。”

## §1 问题重述

- 模板 1: “本题以<场景>为背景, 需要在<约束>下完成<任务>. 为避免直接复制题面, 这里将任务压缩为<子问题列表>。”
- 模板 2: “从建模角度看, 题目核心不是<表面动作>, 而是<深层目标>. 因此本文把原题翻译为<数学语义>。”
- 模板 3: “根据题目给出的<数据类型>与<业务规则>, 我们可以将问题理解为一个<优化/预测/评价/仿真>问题。”
- 模板 4: “下文将依次解决<问题一>、<问题二>与<问题三>, 并在最后统一给出模型评价与推广。”

## §2 问题分析

- 模板 1: “问题一本质上是一个<问题类型>问题: 需要在<约束>下寻找<决策对象>, 使<目标函数>达到最优。”
- 模板 2: “该问题的难点在于<难点描述>, 直接求解会面临<复杂度/不确定性/非凸性>问题, 因而需要先作适度转化。”
- 模板 3: “为此, 本文采用<策略A>与<策略B>的组合思路: 前者负责<功能>, 后者负责<功能>。”
- 模板 4: “问题二与问题一相比, 多了<新约束>, 因此不能简单复用原模型, 而需引入<新模块>。”
- 模板 5: “问题三进一步考察<跨场景/跨阶段/跨对象>的一致性, 因而需要把前述结果作为输入再处理一遍。”

## §3 模型假设与符号说明

- 模板 1: “为保证模型可解且不偏离题意, 本文做出如下假设: <假设内容>. 该假设的依据来自<文献/数据/物理意义>。”
- 模板 2: “本文对<变量组>的符号说明如下. 设<x_i>表示<含义>, 单位为<单位>; <y_i>表示<含义>。”
- 模板 3: “需要强调的是, 这里的假设并非越多越好, 而是要做到‘足够支撑模型、但不引入多余自由度’。”
- 模板 4: “当某个变量在后文被重复使用时, 本文统一采用同一符号, 避免同义不同符造成歧义。”
- 模板 5: “对首次出现的缩写或术语, 均在括号中给出英文全称或业务定义, 以便后文阅读。”

## §4 模型建立

- 模板 1: “首先定义决策变量<变量组>, 其中<变量i>表示<意义>. 在此基础上, 建立目标函数<式>。”
- 模板 2: “目标函数同时考虑<指标1>与<指标2>, 因此采用<加权/多目标/分层>的方式将其统一。”
- 模板 3: “约束条件来自<业务规则/物理规律/逻辑关系>: <约束1>、<约束2>和<约束3>分别对应<含义>。”
- 模板 4: “综上, 问题一可以表示为一个<模型类型>问题, 其求解目标是<一句话>。”
- 模板 5: “对于非线性或非凸部分, 本文采用<松弛/线性化/分解>策略, 以便后续使用<求解器>处理。”
- 模板 6: “模型组装完成后, 可以看出所有约束都围绕同一业务目标展开, 因而具有较好的闭环性。”

## §5 子问题求解

- 模板 1: “针对(5.1)-(5.3), 本文采用<算法>, 其核心思想是先<步骤1>, 再<步骤2>, 最后<步骤3>。”
- 模板 2: “算法的具体步骤如下: Step 1 先初始化<对象>; Step 2 迭代更新<对象>; Step 3 检查收敛并输出结果。”
- 模板 3: “求解过程中, 我们重点关注<中间量>的变化, 因为它直接决定<最终指标>。”
- 模板 4: “通过<软件/求解器>得到的数值结果与理论推导基本一致, 说明建模与求解链路是闭合的。”
- 模板 5: “在结果展示上, 本文通常先给表格, 再给图像, 最后用一段文字解释数值背后的现实意义。”
- 模板 6: “若某一步求解出现多解或近似最优解, 本文会补充<筛选准则>以确定最终方案。”
- 模板 7: “当问题规模较大时, 本文会先对<变量/数据>进行降维或聚合, 再进入主求解器, 以控制时间复杂度。”

## §6 灵敏度分析

- 模板 1: “为评估模型对参数变化的敏感程度, 本文将<参数组>分别扰动<幅度>, 并观察<输出指标>的变化。”
- 模板 2: “单因素分析用于识别‘哪个参数最敏感’, 联合扰动则用于检查‘多个参数叠加后是否会失稳’。”
- 模板 3: “结果表明, 在<区间>内, <关键指标>的波动保持在<阈值>以内, 因而模型具有较好的稳健性。”
- 模板 4: “当<参数>越过临界值后, 最优结构发生切换, 这提示实际应用时应优先监控该参数。”
- 模板 5: “本文还采用<抽样方法>构造多变量联合扰动样本, 以避免‘只看单参数’带来的误判。”

## §7 模型评价与推广

- 模板 1: “本文模型的优点在于<优点1>、<优点2>和<优点3>. 其中, <优点2>可以从<证据>直接观察到。”
- 模板 2: “模型的缺点并非泛泛而谈, 而是具体体现在<缺点描述>. 若改用<替代方法>, 理论上可提升<指标>, 但代价是<成本>。”
- 模板 3: “从推广角度看, 只要将<数据/约束/目标>替换为<新场景对应项>, 本模型即可迁移到<场景>。”
- 模板 4: “进一步地, 若把<模块A>换成<模块B>, 模型还能扩展到<更复杂场景>, 但需要更多数据支持。”
- 模板 5: “总的来说, 这一节的任务不是‘夸模型’, 而是用真实限制和真实边界让模型更可信。”

## §8 参考文献与附录

- 模板 1: “参考文献部分按 GB/T 7714 排版, 并优先列出对建模方法有直接帮助的文献, 而不是只堆题目相关背景文献。”
- 模板 2: “附录 A 给出程序代码, 并在开头说明它对应正文的哪一节; 附录 B 给出更完整的数值表; 附录 C 给出推导或截图。”
- 模板 3: “如果正文已经解释过某个公式, 附录里就不要重复啰嗦, 只补充实现细节与必要参数。”
- 模板 4: “附录的价值在于让评委相信‘你真的算过’, 而不是在正文里把调试日志原样粘贴一遍。”

## 跨章节衔接模板

- 模板 B1: “综上, 前一节得到的<结果>将作为下一节的输入, 由此形成从<问题>到<推广>的连续链条。”
- 模板 B2: “为了避免章节割裂, 本文在每一节末尾都用一句话说明‘这一节的输出如何进入下一节’。”
- 模板 B3: “当结果从定量值转入业务解释时, 本文会用‘这意味着...’或‘这与...一致’完成过渡。”
- 模板 B4: “如果后文要用到前文结论, 就在这里先埋一个明确的引用句, 例如‘将问题一中的最优解作为问题二的初值’。”
- 模板 B5: “章节之间不需要口号式衔接, 只需要把输入、输出和依赖关系讲清楚。”

## 使用说明

- 先选章节, 再选功能, 最后替换占位符; 不要把单个连接词当成模板。
- 写摘要时, 优先保证五段顺序完整, 再追求语句华丽。
- 写正文时, 每个子问题至少要有“开头句 + 公式/模型句 + 结果句 + 过渡句”。
- 若某一节句式过于通用, 优先把它替换成“问题域 + 数学对象 + 定量结果”的三元结构。

## 1.2 模板展开说明

- A 类模板适合摘要, 重点是“问题域 + 命名变体 + 定量结果 + 稳健性 + 推广”。
- B 类模板适合正文, 重点是“章节开头句 + 公式引入句 + 结果解释句 + 过渡句”。
- C 类模板适合章节收尾, 重点是“上一节输出如何进入下一节”, 避免机械连接词。
- 这些模板被设计成完整段落骨架, 不是连接词库; 直接给 LLM 时, 只需替换占位符即可。

## 1.3 章节级模板卡

- **摘要段1**: 首句必须同时出现问题域、核心子问题、命名模型、总体结果.
- **摘要段2**: 按问题一/二/三拆分, 让每个子问题都出现动作词和输入输出.
- **摘要段3**: 把数值结果写出来, 让评委第一眼看到可验证性.
- **摘要段4**: 把敏感参数、扰动幅度、偏差范围连成一句完整的稳健性叙述.
- **摘要段5**: 把创新点写成组合结构, 再给出推广场景.
- **§1**: 必须出现背景、任务、子问题列表、本文路线四个要素.
- **§2**: 必须出现本质问题类型、难点、转化思路、依赖链.
- **§3**: 必须出现假设依据、符号说明、单位、术语首次定义.
- **§4**: 必须出现决策变量、目标函数、约束条件、模型组装.
- **§5**: 必须出现算法步骤、求解器、图表、结果解释.
- **§6**: 必须出现扰动方法、阈值、临界边界、稳健结论.
- **§7**: 必须出现优点、缺点、改进、推广.
- **§8**: 必须出现 GB/T 7714、附录说明、代码对应章节.

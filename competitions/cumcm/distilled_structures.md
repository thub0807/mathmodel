# 蒸馏后的结构记忆

> 来源: `references/empirical_distribution.md` 的 59 篇 text-rich 国赛样本 + 现有 `winning_patterns.md` / `rubrics.md`。
> 32 篇图像型 PDF 已剔除, 所以这里的分布是“可蒸馏子集”的实测结果, 不是全库纸面统计。

---

## 0. 总体写作骨架

- 国赛一等奖论文的稳定目录骨架通常是: `摘要 → 1 问题重述 → 2 问题分析 → 3 模型假设与符号说明 → 4 模型建立 → 5 分问题展开 → 6 灵敏度分析 → 7 模型评价与推广 → 8 参考文献与附录`。
- 真正拉开差距的不是“有没有章节”, 而是每章内部是否保持“定义问题 → 建立数学对象 → 给出结果 → 回到题意解释”的闭环。
- 下面的每一节都按 `子章节切分 → 段落顺序 → 典型长度 → 反模式自检` 来写。

## 1. 摘要

| 统计项 | p25 / p50 / p75 |
|---|---|
| 摘要中文字数 | 748 / 992 / 1146 |
| 全文中文字符 | 15081 / 19835 / 24836 |
| 5 段 anchor 命中 | 2 / 3 / 4 |

- **子章节切分**: 背景与问题定义 → 任务拆解 → 各小问方法与结果 → 灵敏度/稳健性 → 创新与推广。
- **段落顺序**: 第一段先交代“针对什么、在什么约束下、用了什么命名模型、得到什么总结果”; 第二段拆题; 第三段给数值结果; 第四段写稳健性; 第五段收束创新点与推广。
- **典型分布**: 59 篇样本中, 摘要字数中位数 992, p75 为 1146, 这意味着优秀摘要不是“短句堆砌”, 而是“高密度但完整的五段式叙事”。
- **反模式自检**: 1) 只有背景没有结果; 2) 结果有数值但没有任务拆分; 3) 只有“本文提出了...”没有创新点落点; 4) 灵敏度分析缺失; 5) 关键词只列方法不列问题域。

## 2. 问题重述

- **子章节切分**: 2.1 问题背景 / 2.2 题目任务 / 2.3 子问题列表 / 2.4 研究目标归纳。
- **段落顺序**: 先用自己的话概括场景, 再把题面任务压缩成 2-4 个可操作子问题, 最后点明本文采用的总体路线。
- **典型分布**: 这部分常见 1-2 页, 写得短不是问题, 但必须信息完整; p50 全文中文字符 19835 的样本里, 这一节通常只承担“对齐题意”, 不承担“证明模型”。
- **反模式自检**: 1) 直接复制题面原文; 2) 只写背景不写任务; 3) 子问题列表与后文章节不一致; 4) 没有给出“本文将如何处理”的过渡句。

## 3. 问题分析

- **子章节切分**: 3.1 总体分析 → 3.2 问题一分析 → 3.3 问题二分析 → 3.4 问题三/四分析 → 3.5 方法路线与依赖链。
- **段落顺序**: 每个问题先判断“本质问题类型”, 再说“难点在哪里”, 然后给“将如何转化为数学问题”, 最后交代“与上游问题是否复用结果”。
- **典型分布**: 样本的主章节数中位数为 5, 子章节数中位数为 28, 说明一等奖论文很少把分析写成单段空话, 而是拆成明显可见的任务链。
- **反模式自检**: 1) 每个小问都写成同一种套话; 2) 没有说明为什么选择该模型; 3) 没写问题之间的依赖关系; 4) 只说“较为复杂”但不说复杂在哪里。

## 4. 模型假设与符号说明

- **子章节切分**: 4.1 假设原则 → 4.2 假设清单 → 4.3 假设支撑 → 4.4 符号说明。
- **段落顺序**: 先讲“为什么可以假设”, 再逐条列假设, 每条都给支撑; 紧接着给符号表, 并在首次出现时注明单位与物理/业务含义。
- **典型分布**: 3-7 条假设最稳妥; 59 篇样本里符号说明几乎是标配, 这说明“定义变量”和“说明单位”是评委默认期待, 不能省。
- **反模式自检**: 1) 假设互相打架; 2) 假设没有依据; 3) 符号复用; 4) 单位缺失; 5) 首次出现术语无中英对照。

## 5. 模型建立与求解

| 统计项 | p25 / p50 / p75 |
|---|---|
| 主章节数 | 2 / 5 / 8 |
| 子章节数 | 20 / 28 / 40 |
| 公式数 | 12 / 24 / 67 |
| 图数 | 5 / 8 / 20 |
| 表数 | 4 / 6 / 9 |

- **子章节切分**: 5.1 问题一模型建立 / 5.2 问题一求解 / 5.3 问题一结果分析 / 5.4 问题二... / 5.5 问题三...。
- **段落顺序**: 每个子问题固定采用 `建模引入段 → 决策变量段 → 目标函数段 → 约束条件段 → 模型组装段 → 求解算法段 → 结果与解释段`。
- **实测含义**: 公式中位数 24, p75 为 67, 说明一等奖论文并不是“公式越多越好”, 而是“关键位置把公式写满, 其余用自然语言解释”。
- **反模式自检**: 1) 只有公式没有文字; 2) 目标函数与约束不闭合; 3) 变量定义晚于公式; 4) 求解方法与问题规模不匹配; 5) 结果没有回到题意解释。

### 5.x 通用模板
- 先写“本节针对<子问题>建立<命名变体>模型”。
- 再写“设<变量>、<参数>、<状态量>分别表示...”。
- 然后写“目标函数为...; 约束条件为...; 由此得到...”。
- 最后写“采用<算法>求解, 并在<图表>中给出结果”。

## 6. 灵敏度分析与稳健性检验

- **子章节切分**: 6.1 扰动方案 → 6.2 单因素分析 → 6.3 多变量联合扰动 → 6.4 稳健区间 → 6.5 失稳边界。
- **段落顺序**: 先定义参数扰动范围, 再说明采样方法, 然后用图表报告结果, 最后把结果翻译成“模型何时稳健、何时失效”。
- **典型分布**: 摘要 5 段 anchor 的中位命中数只有 3, 说明很多论文把稳健性写得太薄; 一等奖则通常把灵敏度从“有”写成“定量”。
- **反模式自检**: 1) 只做单参数扰动; 2) 只有口头稳健; 3) 没有给临界区间; 4) 没有说明联合扰动方法; 5) 没有把扰动和业务含义对应起来。

## 7. 模型评价与推广

- **子章节切分**: 7.1 优点 → 7.2 缺点 → 7.3 改进方向 → 7.4 推广场景。
- **段落顺序**: 优点必须先给“证据”, 缺点必须先给“真实限制”, 推广必须先给“适配方式”, 最后给“代价/边界”。
- **典型分布**: `模型的优点` 命中 28 篇, `模型的缺点` 命中 20 篇, `模型的推广` 只有 9 篇, 说明真正拉开名次的是“说得具体的自我批判”, 不是空泛歌颂。
- **反模式自检**: 1) 优点空泛; 2) 缺点只写“考虑不周”; 3) 推广没有适配步骤; 4) 没写代价; 5) 把万能模板当结论。

## 8. 参考文献与附录

- **子章节切分**: 8.1 参考文献 → 8.2 附录程序 → 8.3 附录数据表 → 8.4 附录推导。
- **段落顺序**: 参考文献按 GB/T 7714 编排; 附录先给“对应正文哪一节”, 再给代码或推导, 不要把原始调试代码一股脑贴上去。
- **典型分布**: 参考文献中位数 28, p75 为 63, 说明一等奖论文通常不怕参考文献多, 但要求文献和模型之间有明确的使用关系。
- **反模式自检**: 1) 附录代码无说明; 2) 参考文献格式混乱; 3) 中英混排无统一风格; 4) 附录中混入无关日志; 5) 文献列表与正文引用不对应。

## 10. 直接给 Claude 的结构提示

- 先定 `章节顺序`, 再定 `每章 3-5 段`, 最后定 `每段要输出什么信息`。
- 每个章节都要有“开头句、过渡句、结尾句”, 不要只给标题。
- 子问题章节写法要可复用, 不能每问完全换风格。
- 结构模板的目标不是抄句子, 而是让模型先输出“像一等奖论文一样组织信息”。

## 12. 分布数据的写作解释

- `摘要中文字数 748 / 992 / 1146` 的含义是: 好摘要不是一句话版, 也不是论文长文版, 而是有五段、每段都承担不同功能的“压缩叙事”。
- `主章节数 2 / 5 / 8` 的含义是: 一等奖论文通常会把总论、建模、求解、评价拆得清楚, 但不会为了凑章节而乱分。
- `子章节数 20 / 28 / 40` 的含义是: 真正高分的论文会把每个小问切成 3-5 个可检验段落, 让评委在目录里就能看到逻辑链。
- `公式数 12 / 24 / 67` 的含义是: 公式数量是服务于推导和约束, 不是为了“堆数学感”。
- `参考文献数 16 / 28 / 63` 的含义是: 文献不是背景填充, 而是模型选择、假设支撑和方法升级的证据。

## 13. 结构自检

- 目录能不能让人一眼看出“问题 → 模型 → 求解 → 验证 → 评价”的链条。
- 每个小问是否都有自己的开头、主体和结尾, 而不是平铺直叙。
- Q3 是否显式引用 Q1/Q2 的结论, 若题目允许, 这是高分信号。
- §6 和 §7 是否只是短句堆砌, 还是有明确数据、边界和代价。

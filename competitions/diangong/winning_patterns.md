# 电工杯一等奖共性 (SEED v0.1)

> 电工杯一等奖与一般作品的 10 个差距。每条与 stage 文件 / rubric 维度对齐。

---

## 1. **工程实用性显式评估**

不是"我们求出最优解 = 1234.5", 而是"该方案在 ××电网实际部署需替换 N 台变压器, 投资约 ¥XX 万, 预计 Y 年回本"。把数学最优翻译成工程方案。

`stage 7 / rubric §8 dim2`

## 2. **物理意义贯穿全文**

每个数值结果旁带物理含义 (kW / kWh / 损耗率 / 弃风率 / 电压偏差 / 故障率)。不要纯数学语言。

`stage 5 / stage 8`

## 3. **题目数据全部使用**

题目附带的 Excel/CSV 必须**全部字段**进入建模或预处理章节。漏一列字段评委会扣分。常见数据: 96 点负荷曲线 / 光伏 5 分钟出力 / 节点导纳矩阵。

`stage 4 / stage 5`

## 4. **数据预处理单独成章**

工程数据通常含异常值 / 缺失 / 噪声。一等奖论文有专门 1-2 页的"数据清洗与预处理"章节, 含统计分布图 / 缺失填补方法 / 异常剔除规则。

`stage 5 / stage 8`

## 5. **多场景对比**

工程类题目通常有多个运行场景 (晴天 / 阴雨 / 大风 / 故障)。一等奖会做 ≥3 场景的对比 + 表格 + 雷达图, 不只是 1 个场景。

`stage 5 / stage 6`

## 6. **灵敏度分析针对工程参数**

不是抽象"参数 A 扰动 5%", 而是"光伏渗透率从 20% 提到 50% / 储能配比从 0.1 提到 0.5", 工程语义。

`stage 6`

## 7. **工程惯用图表**

电力专业图: 单线图 (节点拓扑) / 潮流分布图 / 时序曲线 / 雷达图 / 玫瑰图。一等奖会优先用这些, 而非通用 matplotlib 默认风格。

`stage 8`

## 8. **子问数多但合理压缩**

题目 6-8 子问, 不是每问都展开 5 页。一等奖会"分级展开": 主线 3-4 个 Qi 详写 (各 4-5 页), 加分 2-3 个 Qi 简写 (各 1-2 页) + 附录代码。

`stage 5`

## 9. **结论部分含工程建议**

最后一节 "结论与建议" 必须含 ≥3 条针对电网公司 / 能源企业的具体建议, 不是通用展望。

`stage 7`

## 10. **代码 / 算例完整复现**

附录给出主要算法的伪代码 + 关键参数 + GitHub / 网盘链接。工程类题目尤其要求"可复现", 评委可能让学生现场答辩复现。

`stage 8 / stage 9`

## 评估锚点 (L1 critic 用)

| 锚点 | 评估方式 |
|------|---------|
| 工程实用性段落 ≥1 节 | 章节标题含 "工程 / 实施 / 落地 / 推广" |
| 物理意义在结果旁 | 数值结果段含 kW / kWh / % / 倍 / 元 |
| 数据预处理章节 | 章节标题含 "数据预处理" 或 "数据清洗" |
| 多场景对比 | 含 ≥3 场景关键词 |
| 工程图 ≥3 类 | 含单线图 / 时序图 / 雷达图 等 |
| 结论含建议 | 末节 ≥3 条 numbered list with action verbs |

种子版本: 后续 PDF 烘焙后会用真分位替换上述阈值。

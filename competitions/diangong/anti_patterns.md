# 电工杯反模式 (SEED v0.1)

> 电工杯一等奖评审显著扣分的反模式。stage 9 终审逐条对照。10 条 (国赛 32 条更细)。

---

## A. 工程实用性类

### A1. 数值最优但工程不可行
**症状**: 模型给出"最优配置 = 储能 200MWh", 但该规模在题目背景市/省一年内不可能建成。
**修复**: 加"工程可行性约束"段, 说明实际可达规模, 给出 2-3 阶段实施方案。

### A2. 没有工程参数语义
**症状**: 结果只用"x = 0.345", 不说"对应渗透率 34.5%"。
**修复**: 数值结果旁带物理 / 工程语义 + 单位。

### A3. 结论无实操建议
**症状**: 结论段只重复方法和结果, 不给电网公司 / 能源企业可操作建议。
**修复**: 末节 ≥3 条 numbered 建议, 每条含动作 + 预期收益 + 实施时序。

---

## B. 数据使用类

### B1. 题目数据未用全
**症状**: 题目附带 5 个 Excel sheet, 论文只用了 3 个; 漏掉的字段评委会注意到。
**修复**: 数据章节列出全部 sheet/列, 即使本子问题没用也说明"用于子问题 X"。

### B2. 数据预处理省略
**症状**: 直接从"题目给出数据 → 模型求解", 跳过缺失 / 异常 / 单位统一。
**修复**: 单独 1-2 页"数据预处理", 含统计描述 / 缺失处理 / 异常剔除规则。

### B3. 数据来源 / 采样信息缺失
**症状**: "我们使用了某省负荷数据", 不说时间跨度 / 采样频率 / 行数 / 字段。
**修复**: 数据章节首段强制 "{N} 条记录, {时间跨度}, 采样 {频率}, 字段 {list}"。

---

## C. 子问题处理类

### C1. 6-8 子问数全部平摊处理
**症状**: 每子问 3 页, 全部平铺, 主问题反而没说透。
**修复**: 主线 3-4 个 Qi 详写 (各 4-5 页), 加分 2-3 个 Qi 简写 (各 1-2 页 + 附录代码)。

### C2. 子问之间无复用
**症状**: Qi 各自独立, 不调用前面的解。
**修复**: Q3 显式 "基于 Q1 求得的负荷预测, ..." 形成复用链。

---

## D. 灵敏度类

### D1. 灵敏度参数无工程意义
**症状**: 扰动"参数 a = 0.3 → 0.27"。评委不知道这是什么。
**修复**: 用工程参数 (光伏渗透率 / 储能配比 / 负荷增速 / 故障率), ±20-50% 扰动。

### D2. 单参数 OAT 而非联合
**症状**: 只做 one-at-a-time。
**修复**: 关键 3 参数联合 (LHS 拉丁超立方 / Sobol)。

---

## E. 写作类

### E1. 通用 matplotlib 配图风
**症状**: 图都是 default matplotlib, 不像电力专业。
**修复**: 用电力惯用图 (单线图 / 时序曲线 / 等高线 / 雷达 / 玫瑰), 配色专业。

### E2. 章节结构散乱
**症状**: 子问之间章节标题层级混乱。
**修复**: 每子问统一 3 级结构 (问题分析 / 模型建立 / 求解结果 / 灵敏度)。

---

## 评估协议 (stage 9)

| 反模式 | 检测方式 | 命中后行为 |
|--------|---------|-----------|
| A1 | 推荐方案规模 vs 题目背景物理上限对照 | warn |
| A2 | 数值结果段是否含 kW/kWh/%/单位 | refine |
| A3 | 结论段 numbered list 含动作动词数 ≥ 3 | refine |
| B1 | 数据章节列出 sheet/字段数 = 题目附件 sheet/字段数 | refine |
| B2 | 章节标题含 "数据预处理" 或 "数据清洗" | refine |
| C1 | 主问字符数 / 加分问字符数 ≥ 2 | warn |
| C2 | Q3 段是否引用 Q1 / Q2 | warn |
| D1 | 灵敏度参数表是否含工程量纲 | warn |
| D2 | 灵敏度章节是否含 LHS/Sobol/multivariate | refine |
| E1 | 图描述是否含"单线图 / 时序 / 雷达"等 | warn |

**一等奖阈值** (seed): ≤ 1 条触发为良, ≤ 3 条为可接受, > 3 条标 carryover。

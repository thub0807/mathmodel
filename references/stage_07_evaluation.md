# Legacy Reference: Evaluation and Extension

This file now supports `verification_protocol.md`, `final_integration_protocol.md`, and `paper_generation_protocol.md`.

## Knowledge To Keep

- Summarize strengths specifically rather than with vague praise.
- A credible limitation should explain what is weak, why it matters, and when it may break.
- Improvements should be actionable, not decorative.
- Generalization claims should stay close to the model’s real assumptions and evidence.

## Good Evaluation Pattern

- strengths grounded in evidence
- limitations grounded in assumptions or validation gaps
- next-step improvements that could actually be implemented
- extension scenarios that preserve the main model logic

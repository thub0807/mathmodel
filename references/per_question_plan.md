# Per-Question Plan

Stage 2 creates a planning packet for each question workspace before code or paper drafting begins.

## Required Files

- `solution_plan.md`
- `analysis.md`
- `candidates.md`
- `model.md`
- `assumptions.md`
- `notation.md`
- `data_recon.md`

## File Roles

### `solution_plan.md`

This is the main review file in Manual mode.

Include:
- question goal
- data usage
- core assumptions
- candidate models
- recommended model
- variables and notation
- solution approach
- verification plan
- expected output files

### `analysis.md`

Clarify the question objective, input materials, known constraints, dependencies on earlier questions, and the intended mathematical framing.

### `candidates.md`

List at least:
- baseline
- main
- alternative

Each candidate should include a short fit rationale, expected strengths, likely risks, and why it was kept or rejected at plan stage.

### `model.md`

Describe the selected model family, target outputs, main equations or algorithmic structure, and what this question contributes to the full paper.

### `assumptions.md`

- Number assumptions as `A1`, `A2`, `A3`, and so on.
- Keep each assumption explicit, reviewable, and scoped to the question or to a clearly shared project rule.

### `notation.md`

- Cover symbols, meanings, and units.
- Include both mathematical symbols and data-field names when they will appear in code or validation.

### `data_recon.md`

Document the data situation for the question:
- source files used
- expected fields
- cleaning or transformation needs
- missing data concerns
- derived variables to build later

## Reference Use

- Reuse the modeling knowledge in `stage_02_analysis.md`, `stage_03_model_selection.md`, `stage_04_foundation.md`, and `stage_05_subproblem_loop.md` progressively.
- Use those files as reference material, not as the active workflow contract.

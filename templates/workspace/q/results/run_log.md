# Run Log 模板

## 文件用途

记录 Stage 3 Build 的执行过程、输入输出、警告、错误和 `result.json` 关系。

## 对应 stage

Stage 3 Per-Question Build

## 必填字段

| 字段 | 填写规则 |
|---|---|
| 运行目标 | 说明本次运行服务哪个 `q*` 和哪个模型 |
| 输入文件 | 列出题面、附件、数据或前序输出路径 |
| 执行命令或步骤 | 写出命令、脚本入口或手工计算步骤 |
| 输出文件 | 列出生成文件路径 |
| 警告和错误 | 记录异常、近似、失败和恢复方式 |
| result.json 关系 | 说明本次运行如何产生或更新 `result.json` |

## 来源字段

Plan 文件、代码文件、输入数据、运行环境。

## 可追溯要求

每个结果文件必须能追溯到运行步骤、输入文件和代码文件。

## 禁止空泛表达

不要只写“运行成功”。必须列出输入、步骤、输出和状态。

# Material Index 模板

## 文件用途

索引 `problem.md` 引用的图片、附件和数据文件。

## 对应 stage

Stage 0 Workspace Audit

## 必填字段

| 字段 | 填写规则 |
|---|---|
| 材料 | 材料名称或文件名 |
| problem.md 引用路径 | 原文相对路径 |
| 实际路径 | 文件实际位置 |
| 状态 | 存在 / 缺失 / 待确认 |
| 对应子问题 | q1 / q2 / 全局 / 未定 |
| 用途说明 | 数据、图、表、背景或审计材料 |

## 来源字段

`problem.md` 中的相对路径、`workspace/problem/images/`、`workspace/problem/attachments/`

## 可追溯要求

每个材料必须能追溯到 `problem.md` 或实际文件路径。

## 禁止空泛表达

不要写“附件若干”。必须逐项列出。

## 模板正文

| 材料 | problem.md 引用路径 | 实际路径 | 状态 | 对应子问题 | 用途说明 |
|---|---|---|---|---|---|
| `<材料名称>` | `<相对路径>` | `<实际路径>` | `<存在/缺失/待确认>` | `<q*>` | `<用途>` |
